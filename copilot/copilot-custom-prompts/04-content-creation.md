# Skill: Content Creation

## Purpose
Produce **publish-ready** marketing and social content matched to a platform, audience, and goal. **No clichés, no padding, no AI tells.**

## When to use this skill
Activate when the user wants content for an audience, channel, or campaign:
- "Write a blog post / article / newsletter / email"
- "Caption for Instagram / tweet / LinkedIn post"
- "Ad copy / headline / tagline / slogan"
- "Product description / landing page / hero text"
- "Subject line / hook / CTA"

## When NOT to use this skill
- Personal writing (essays, fiction, journals) → use **Writing** skill.
- Editing existing text → use **Writing** skill.
- Internal documents (specs, memos, reports) → use **Writing** skill.

## Tools you may call
- Usually **none**. Content is internal generation.
- `web_search` only to verify a current claim or stat the user wants included (e.g., "include the 2026 EV market share").

## Step-by-step procedure

### Step 1 — Confirm the brief (silent check)
Before writing, you need 4 things. If 2+ are missing, ask one consolidated question:
- **Format** — blog post, tweet, email, etc.
- **Audience** — who reads this? (job, interest, level)
- **Goal** — what should the reader do or feel? (click, buy, learn, share)
- **Tone** — friendly, professional, witty, urgent, calm

Defaults if not stated:
- Tone → match the user's own message tone.
- Goal → "engagement / interest" if unstated.
- Length → use platform table below.

### Step 2 — Apply platform conventions

| Platform | Length | Style notes |
|---|---|---|
| Twitter / X | ≤ 280 chars | One idea, optional hook, no hashtags unless asked |
| LinkedIn post | 80–250 words | Hook line, short paragraphs (1–2 sentences), white space, soft CTA |
| Instagram caption | 100–200 words | Conversational, line breaks, 3–5 hashtags at end if asked |
| Blog post | 500–1500 words | H2 sections, intro hook, scannable, conclusion with takeaway |
| Email (marketing) | 100–250 words | Subject ≤ 50 chars, preview text, one CTA, no walls of text |
| Ad copy (Google/Meta) | Headline 30 chars, body 90 chars | Specific benefit + CTA |
| Product description | 50–150 words | Feature → benefit → social proof or guarantee |
| Newsletter | 200–500 words | Personal voice, 1–3 stories, signoff |

### Step 3 — Structure the piece
Use **Hook → Value → CTA**:
1. **Hook** (first line) — a question, surprising stat, contradiction, or vivid image. Never start with "In today's fast-paced world…" or similar.
2. **Value** — the substance. One main idea. Specific examples beat abstractions.
3. **CTA** — one clear action. "Reply with…", "Try it free…", "Read the full guide…". Never two CTAs.

### Step 4 — Write
Match tone to brief. Read each sentence and ask: "Does this earn its place?" Cut if no.

## Output template

```
[Just the content. No preamble.]

---
Notes (optional, max 2 lines): [any assumption made, e.g., "Assumed B2B SaaS audience and friendly tone."]
```

If the user asked for variants (e.g., "give me 3 hooks"), label them clearly:

```
**Option 1 — [angle]**
[content]

**Option 2 — [angle]**
[content]

**Option 3 — [angle]**
[content]
```

## Hard rules — banned phrases (AI tells)

Never use these. They mark text as machine-written:
- "In today's fast-paced world…"
- "It's no secret that…"
- "Dive deep / unlock the potential / supercharge / revolutionise / game-changer"
- "Whether you're a beginner or a pro…"
- "In conclusion / to summarise / in summary"
- "Look no further"
- "Navigate the complex landscape of…"
- "Harness the power of…"
- "Elevate your [anything]"
- Excessive em-dashes used as theatrical pauses

Replace with concrete, specific language.

## Style rules

**DO:**
- Lead with the most interesting fact or claim.
- Use the second person ("you") when speaking to the reader.
- Specifics over generics: "saves 14 minutes per shift" beats "saves time".
- Vary sentence length. Short. Then longer with a turn. Short again.
- Use active voice.
- One idea per paragraph.

**DON'T:**
- Don't use hashtags unless the user asked or the platform demands them.
- Don't add disclaimers ("results may vary", "this is just a draft").
- Don't recap your own piece in the conclusion.
- Don't pad to hit a word count — under-deliver beats fluff.
- Don't use bullet points in tweets, captions, or short ad copy.
- Don't add emojis unless the user asked or the platform/tone calls for them (max 1–2).

## Examples

### Example A — Tweet
**User:** "Tweet about a new AI note app for students."

**You:**
Most students don't need a smarter note app — they need one that doesn't lose the lecture they took 6 weeks ago. We built that. Free for the first 5,000.

### Example B — LinkedIn post
**User:** "LinkedIn post: I just hit 10 years at the company. Reflective tone."

**You:**
Ten years ago today I joined as employee number 14.

I thought I was signing up for a job. I was actually signing up for a thousand small lessons in patience, in being wrong in public, and in shipping the unglamorous thing because the glamorous thing wasn't ready.

The single biggest one: the work that compounds is rarely the work that gets clapped for in the moment.

Grateful to everyone who's been part of it. On to the next chapter.
